◆ isEnabled()

virtual LIBAPI bool LunarSimulation::HlaInteractionManager::isEnabled ( Interaction  interaction)
pure virtual

Check if an interaction is actually enabled when connected.

An interaction may be disabled with the setEnabled(false, ...) function, or it may be configured as if available in the FOM by the setEnabledIfAvailableInFom(...) function and then not available in the FOM at runtime.

This method should only be used when connected, it will always return false when not connected.

Parameters
interactionthe interaction to check
Returns
true if connected and the interaction is enabled.

executionModerepresents the execution_mode from the FOM.
Description from the FOM: The run mode requested. There are only 3 valid Mode Transition Request (MTR) mode values: MTR_GOTO_RUN, MTR_GOTO_FREEZE, MTR_GOTO_SHUTDOWN. Of these three valid mode requests, only 7 combinations of current execution mode and requested mode are valid: 1. EXEC_MODE_UNINITIALIZED -> EXEC_MODE_SHUTDOWN 2. EXEC_MODE_INITIALIZED -> EXEC_MODE_FREEZE 3. EXEC_MODE_INITIALIZED -> EXEC_MODE_SHUTDOWN 4. EXEC_MODE_RUNNING -> EXEC_MODE_FREEZE 5. EXEC_MODE_RUNNING -> EXEC_MODE_SHUTDOWN 6. EXEC_MODE_FREEZE -> EXEC_MODE_RUNNING 7. EXEC_MODE_FREEZE -> EXEC_MODE_SHUTDOWN
Description of the data type from the FOM: Mode Transition Request (MTR) transition values. This enumeration is used to request a specific mode transition. However, not all mode transition requests are accepted for any given Run Mode. See mode transition validation table in the Space Reference FOM documentation.
