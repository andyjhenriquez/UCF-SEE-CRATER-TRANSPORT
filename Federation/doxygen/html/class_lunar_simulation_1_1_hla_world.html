virtual LIBAPI HlaLogicalTimePtr LunarSimulation::HlaWorld::advanceToNextEvent ( )
pure virtual

Advance the logical time to the next event or, if no events are available, to the next frame.

This will perform the Next Message Request Available HLA call. This method support 0-lookahead so that messages may be sent with the same logical time as the federate has advanced to.

Note that this method is an implicit evokeAll() in manual evoke mode, to verify that all notifications for the previous time has been processed before advancing to the next event.

Returns
the logical time for the new, now current, frame.

◆ enableTimeManagement()

virtual LIBAPI bool LunarSimulation::HlaWorld::enableTimeManagement ( )
pure virtual

Enable time management.

The federate will become regulating and/or constrained based on the settings objects supplied on create. For this method to have any effect, the tuning flag ENABLE_TIME_MANAGEMENT_ON_CONNECT must be false.

Note that this method will do nothing if constrained and regulating are false in settings. Note that this method will do nothing if time management is already enabled, i.e. calling this multiple times has no effect.

Returns
true if time management was enabled, otherwise false
Exceptions
HlaRtiExceptionif there are RTI related problems
HlaInternalExceptionIf there is an internal error
HlaInvalidLogicalTimeExceptionif the logical time is not valid.
HlaNotConnectedExceptionif the federate was disconnected during initialization

◆ evoke()

virtual LIBAPI void LunarSimulation::HlaWorld::evoke ( )
pure virtual

Evokes a single notification from the notification queue.

The notification will be called from this thread. This method will return after the notification has returned. If there are no pending notifications, this method will return immediately.

If this HlaWorld is in auto evoke mode, this method will be a no op.

Exceptions thrown in the user notification code will be posted to the ExceptionListener.

◆ evokeAll() [1/2]

virtual LIBAPI void LunarSimulation::HlaWorld::evokeAll ( )
pure virtual

Evokes all pending notifications from the notification queue.

The notifications will be called from this thread, one at a time. This method returns when the notification queue is empty.

Exceptions thrown in the user notification code will be posted to the ExceptionListener.

If this HlaWorld is in auto evoke mode, this method will be a no op.

◆ evokeAll() [2/2]

virtual LIBAPI void LunarSimulation::HlaWorld::evokeAll ( unsigned long long  millis)
pure virtual

Evokes notifications for at least millis milliseconds with this thread.

If all pending notifications are processed before millis milliseconds have elapsed, the current thread is suspended until either some more notifications arrive, or the time has elapsed.

Note that this method could return even when there are pending notifications left.

If this HlaWorld is in auto evoke mode, this will suspend the current thread until the time has elapsed.

Exceptions thrown in the user notification code will be posted to the ExceptionListener.

Parameters
millisThe minimum time, in ms, to process notifications.
